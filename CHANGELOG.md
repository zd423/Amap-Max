# 更新日志

本文档用于记录 AMap Companion 的更新日志。

## 2026-06-07 低透明度可读性优化

- 所有悬浮窗文字新增自动对比描边：根据当前文字颜色自动选择黑色或白色轮廓，改善低背景透明度、浅色地图和复杂车机桌面上文字融入背景的问题。
- 主屏与仪表副屏使用相同的文字描边逻辑，切换自动文字颜色或不同 UI 样式后会重新应用轮廓。
- 无蓝色背景的车道图标新增黑色轮廓，包括高德车道位图和程序绘制的兜底箭头；带蓝色背景的车道栏保持原有样式，避免重复描边。
- 继续优化无法识别的特殊车道兜底图标尺寸与资源体积。

## 2026-06-06 特殊车道兼容与桌面启动直达

- 新增无法识别的特殊车道兜底图标：高德广播中的未知或扩展车道编码不再显示为空白，统一使用特殊车道提示图。
- 特殊车道提示图单独缩小至普通车道图标的约 85%，改善与常规车道并排显示时的视觉比例，普通车道尺寸保持不变。
- 新增“桌面启动时直接进入目标应用”选项：开启后，从桌面点击 AMap Companion 会直接打开用户选择的目标应用。
- 桌面直达不会影响伴侣设置入口；点击主屏悬浮窗仍会进入 AMap Companion 设置页面。
- 桌面直达时会按现有主屏、副屏和广播自动显示选项恢复伴侣服务，不改变用户已经选择的悬浮窗显示策略。
- 桌面直达新增快速返回设置规则：首次点击桌面图标进入目标应用，30 秒内再次点击则进入伴侣主界面；后续点击会重新开始一轮判断。
- 修复主屏悬浮窗首次显示后 4 秒内不可触摸的问题，现在窗口出现后即可单击进入伴侣设置；副屏启动保护逻辑保持不变。
- “选择目标应用”改为先显示弹窗、后台扫描应用列表，避免枚举应用和排序时阻塞主界面；应用图标增加缓存，改善列表滚动流畅度。

## 2026-06-06 广播解析重构、主副屏稳定性与卡片 UI 完善

- 重构高德广播处理架构：红绿灯、车道、服务区、偏好设置、日志收集、Markdown 渲染和 UI 样式注册拆分为独立模块，统一数据解析入口，降低不同 UI 重复解析和后续扩展新样式的成本。
- 经典 UI、卡片 UI、灵动岛、灵动岛（测试）和新 UI 改为独立布局构建；保留经典 UI 原有车道样式，修复重构期间不同 UI 车道布局互相覆盖、导航道路信息缺失等问题。
- 修复启用副屏镜像后切换 UI 可能闪退或显示旧布局的问题：切换样式时会先完整销毁旧仪表镜像，再按新样式重建，主屏与副屏显示内容和格式保持一致。
- 修复反复点击“关闭伴侣服务/启动伴侣服务”后出现多层悬浮窗残留的问题；关闭服务现在会同步移除主屏和副屏窗口，并取消初始化、副屏重试、超速闪烁等全部延迟任务。
- 统一无导航数据时的初始状态，各 UI 显示“待接收导航/巡航信息”；高德退出或停止广播后会清理道路、方向、车道、红绿灯和电子监控旧数据，不再长期残留上一段导航内容。
- 卡片 UI 继续优化：修复导航信息行重叠和副屏底部空白行；巡航顶部调整为“方向、道路名称、电子监控”，方向与道路名紧贴，道路名与电子监控保留适当间距。
- 修复限速和限速拍照图标显示：三位数限速值可完整容纳在圆形标识中；限速拍照使用纯限速圆显示数值，不再与摄像头图案重叠。
- 灵动岛 ETA 信息列加宽，“余 xxx km”固定单行显示；新 UI 在相同悬浮窗比例下适当缩小，改善与其他样式面积差异过大的问题。
- 经典 UI 新增高速服务区信息开关，并支持同一广播中的多个服务区；服务区名称与距离按行显示，广播清空或导航退出后同步清除残留。
- 调整超速边框策略：普通超速可选蓝绿色 5 秒闪烁/20 秒停留，超速 10% 可选黄色 5 秒闪烁/10 秒停留，超速 20% 强制红色持续提醒。
- 删除仅用于开发过程的 `重构.md`，保留面向维护者的模块接口和源码结构。

## 2026-06-01 卡片UI布局优化与全样式超速警示

- 电子眼摄像头图标新增限速数值叠加：经典 UI、Dashboard 和卡片 UI 的测速类摄像头会在图标上方显示红色限速数字，与灵动岛全功能模式逻辑一致。
- 卡片 UI 巡航布局重构：第一行固定 32dp、第二行固定 35dp，道路名称与方向文字交换位置，所有元素在行内垂直居中；巡航双行模式面板总高从 68dp 调整为 76dp。
- 卡片 UI 导航布局与巡航统一：导航面板总高固定为 76dp，转向图标从 36dp 放大到 44dp，距离文字从 13sp 放大到 16sp，道路名和 ETA 文字从 12sp 放大到 15sp，导航第一行固定 30dp、第二行固定 40dp。
- 修复卡片 UI 巡航第二行无内容时无法立即折叠为单行的问题，集群副屏同步增加行内容判断逻辑。
- 修复全样式悬浮窗宽度自适应问题：内容变宽后 `stabilizePanelSize` 会强制通知 WindowManager 更新窗口尺寸，避免多车道+红绿灯+电子狗等元素被挤出可视区域或错行显示。
- 新增全样式超速警示功能：当车速超过道路限速时，悬浮窗边缘显示 5dp 彩色边框并以 2Hz 频率闪烁；超过限速显示黄色、超过 10% 显示红色、超过 20% 显示深红色；车速回落至限速以下或限速消失后自动取消警示。
- 车速追踪从 `updateModeFromExtras` 提升到 `handleBroadcast` 公共层，所有 KEY_TYPE 的广播均可更新当前车速，巡航和导航模式均支持实时超速检测。
- 车速数据来源于高德广播的 `CUR_SPEED` / `SPEED` 字段，限速数据来源于 `LIMITED_SPEED` / `CAMERA_SPEED` 字段。

## 2026-05-31 灵动岛全功能模式

- 合入 [#13](https://github.com/zuo-qirun/amap-companion/pull/13) 的“灵动岛（全功能）”样式，新增导航转向信息与 ETA 轮换显示，保留旧 UI、灵动岛（紧凑）、灵动岛（测试）和新 UI。
- 导航状态下，全功能模式会在同一区域按 10 秒转向信息、5 秒 ETA 信息轮换显示；转向信息包含图标、剩余距离和待转入道路名，ETA 信息包含剩余距离和预计到达时间。
- 统一全功能模式主屏/副屏的信息列宽度，转向信息列和 ETA 列保持同等宽度，减少广播刷新时悬浮窗横向跳动。
- 道路名称改回系统原生跑马灯，并保留“文本不变不重新设置”的防重置逻辑，修复自绘跑马灯在信息列中来回快速闪动的问题。
- 修复全功能模式道路名可能不显示的问题：ETA 广播更新道路名后会同步刷新导航道路名，避免转向广播阶段还未拿到路名导致空白。
- 巡航状态下车头方向改为粘性显示：普通状态广播不再清空方向缓存，无新方向数据时保留上一帧，避免“车头：--”与真实方向来回闪动。
- 副屏/仪表同步支持全功能模式的转向、ETA、巡航道路名与车头方向显示。

## 2026-05-28 灵动岛紧凑模式

- 新增“灵动岛（紧凑）”悬浮窗样式，并保留原有 UI、灵动岛、新 UI 和副屏显示逻辑。
- 紧凑模式导航改为胶囊布局：左侧显示转向图标、剩余距离和待转入道路名，车道、红绿灯和电子眼在同一行紧凑展示。
- 紧凑模式巡航在道路名称下方新增“车头：方向”，方向会缓存最近有效值，避免高德重复广播时突然显示/消失。
- 紧凑模式车道信息仅在该模式下启用公共裁剪和 2dp 间隙，普通 UI、灵动岛和新 UI 的车道显示回滚到原仓库逻辑。
- 优化紧凑模式车道尺寸随悬浮窗缩放的比例，避免 100% 及放大后车道箭头过大。
- 紧凑模式红绿灯导航状态下将秒数放入灯圈内，巡航状态保持灯圈内显示方向箭头、下方显示倒计时。
- 电子眼提示改为复用固定子视图并延迟清除，减少公交道、压线、红绿灯等提示反复出现/消失；限速类电子眼会把限速值填入圆圈。
- 副屏/仪表的灵动岛紧凑布局同步主屏信息，补齐道路名、车头方向、车道、红绿灯和电子眼显示。

- 优化导航红绿灯方向兼容：导航态 `dir=0` 不再直接视为掉头，黄灯 3 秒会延续上一阶段绿灯/红灯记录到的方向，避免黄灯尾秒误显示掉头箭头。
